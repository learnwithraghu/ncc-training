# Exercise: System Troubleshooting

## Goal
Practice checking running processes and system activity.

## Tasks
1. View your disk usage with `df -h`.
2. List the top memory-consuming processes with `top` or `ps aux`.
3. Find one long-running process and note its PID.
4. Start a simple background command such as `sleep 300`.
5. Use `ps` to confirm the process is running.
6. End the process with `kill`.

## Check Yourself
- You should be able to explain what a PID is and why `kill` needs one.
