# Exercise and Solution Folder Plan

## Goal

Add a consistent practice structure to the training modules so each section has a clear place for learner exercises and a matching place for instructor solutions.

## Proposed Structure

Each numbered module should use the same pattern:

```text
module-name/
  exercise/
    easy/
    medium/
    hard/
  solution/
    easy/
    medium/
    hard/
```

## Content Approach

- Aim for about 10 exercises per section when the module depth supports it.
- Keep the difficulty split simple and predictable: easy, medium, and hard.
- Make the solution folders mirror the exercise folders so I can demonstrate answers live without confusion.
- Keep the naming easy to follow during training.

## Scope

Start with the numbered course folders, especially the main instructional sections from 01 through 11.

If a module already has a natural hands-on area, the exercise and solution structure should fit that module cleanly without breaking the current guide flow.

## Teaching Flow

1. Show the exercise on screen.
2. Let the learners implement it.
3. Review the solution folder after the attempt.
4. Use the same folder layout every time so the pattern becomes familiar.

## Implementation Notes

- Preserve the existing guide and README structure.
- Add exercise and solution content later, one module at a time.
- Use the first few modules as templates before rolling the pattern across the full repository.
- Keep the file and folder names consistent across all modules.

## Next Step

Once this structure is approved, start creating the folders and populate the first set of exercises and solutions in the easiest module first.
